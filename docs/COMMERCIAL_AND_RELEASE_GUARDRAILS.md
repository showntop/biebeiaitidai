# 商业化与正式发布护栏

## 当前默认状态

商业化配置默认关闭，广告位 ID 为空。代码库不包含插屏入口，也不会因为平台广告不可用而阻断复活、重试、下一关或每日挑战。

只有运行时显式注入 `__BRAATN_COMMERCIAL_CONFIG__`，并同时满足以下条件才允许请求激励视频：

- `enabled=true`
- 有合法 `rewardedAdUnitId`
- placement 在白名单内
- 玩家主动点击
- 未超过单会话上限
- 已过冷却时间

当前只接入“失败复活”这一有真实产品语义的 placement；默认关闭时仍是免费复活。未完成奖励货币设计前，不接“结算翻倍”；未完成挑战次数设计前，不接“额外挑战”。

## 运行监控

`RuntimeMonitor` 采集：

- 未捕获异常 / Promise rejection
- 资源加载失败
- 单帧超过 1.5 秒的卡死信号
- 前后台切换

只记录错误类别和最多 160 字的脱敏短消息；URL 被替换，openid 被遮盖，不采集完整堆栈或用户身份。

## 上线前人工门槛

- 在微信后台配置并审核激励视频广告位，再通过灰度配置开启。
- 补齐隐私政策、数据用途、保存周期、删除方式和客服渠道。
- iPhone 刘海屏/小屏、低端安卓、弱网、离线、后台切回各完成一局。
- 验证广告完成/跳过/失败/无库存四个分支，奖励只发一次。
- 检查职场文案、分享图、广告奖励描述与微信审核规范。
- 先灰度比较留存、复活率、重试率；没有正向结果不扩大广告频次。

这些需要微信后台、真实广告位和真机，不能由本地代码构建替代。
