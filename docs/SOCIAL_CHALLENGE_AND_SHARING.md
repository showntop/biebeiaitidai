# 传播闭环：战报图与同局挑战

P5 不做空排行榜，先验证“玩家是否愿意晒一个有趣结果，并让朋友接同一局”。

## 战报图

`SocialChallenge.buildSharePayload()` 生成平台无关的战报模型，`ShareBridge` 再把它画成 600×480 暖纸质海报。海报有四类：

- 猎杀反杀
- 生存守住
- 红区/复活翻盘
- 失败但差一点守住

指标只使用正向表达：坚守时间、最高连击、Perfect 数、段位和坚守天数。战报图不出现“失误率”“失败次数”等越小越好的比较项。

微信小游戏优先使用离屏 Canvas 临时图片和 `wx.shareAppMessage`；Web 优先使用系统 Web Share，缺失时复制挑战链接。所有能力失败都只降级，不阻断结算、重试或下一关。

## 同局挑战

- 好友挑战保存当前关卡与实际 seed，分享回流后按钮变为“接受好友挑战”。
- 每日挑战由 `YYYY-MM-DD` 确定关卡与 seed，同一天所有设备完全一致。
- 挑战短码带 16 位校验，非法、篡改、越界或超长参数直接拒绝。
- 好友/每日挑战可进入未解锁关，但挑战结算不会解锁主线、增加三星或修改段位。
- 重试继续使用同一个 seed，保证任务流可比较。

## 数据事件

- `share_open`
- `share_result`：shared / copied / canceled / unavailable
- `challenge_start`：daily / friend

只有真实使用数据证明战报分享成立后，才进入排行榜、群挑战或更重的社交能力。
