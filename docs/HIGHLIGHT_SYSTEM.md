# AI 人格与高光系统

高光由纯规则层 `HighlightSystem` 统一判定，局内动效、AI 表情、事件吐槽、结算称号、埋点和后续分享卡只消费同一个 `Highlight` 事件。

## 高光分级

- 轻高光：首次 Perfect、三连击、生存通关、失败局叙事锚点。
- 强高光：五连击、危险区翻盘。
- 封面高光：连续三次 Perfect、Boss 门口拦截、猎杀通关、复活翻盘。

同种高光每局只触发一次。结算选择最高等级、同等级最后发生的一项作为本局称号；即使玩家失败且没有有效命中，也会生成一句克制的“差点保住工位”，避免空白结算。

## AI 六种清晰状态

- 装忙：Boss 进入队列。
- 得意/紧张：认可度进入不同风险区。
- 慌张：危险、复活翻盘。
- 捂脸：连续 Perfect 或 Boss 极限拦截。
- 死机：五连击。
- 被老板叫走：猎杀式通关。

表现层使用局部状态牌、角色形变和短吐槽，不使用整屏红黄闪烁。每次事件区只保留一条最高优先级文案，减少信息噪声。

## 数据口径

- 即时事件：`highlight { id, tier }`
- 局末聚合：`highlightCount / highlightIds / highlightTitle`

这三个字段可直接作为 P5 战报图与每日挑战比较数据，不需要重新推导。
