# 《别让AI替代你》关卡节奏 / IAA变现 / 社交裂变方案

> 承接核心玩法策划与开发计划，聚焦"多关卡的长线运营层"设计：关卡节奏怎么排、广告怎么插不伤体验、社交裂变怎么做才合规又有效。三件事全部服务同一个目标——**上头（持续想玩）、顺畅（不被打断）、爽感（每个节点都有正反馈）**。

---

## 0. 三个体验原则怎么落地（先说结论）

| 原则 | 具体保证手段 |
|---|---|
| **上头** | 锯齿状难度曲线（松-紧交替）+ 道具/机制错峰解锁 + 每日挑战独立轨道 + 好友排行攀比 + 可变奖励（宝箱/随机加成） |
| **顺畅** | 广告只在"自然停顿点"出现、失败重试不重新加载关卡、分享永远是可选加成不阻断主路径、启动/关卡切换控制在可感知延迟以下 |
| **爽感** | 每关有明确星级反馈、结算页数据可视化战报、连续Perfect/极限反杀有专属演出、分享素材本身自带"晒战绩"的满足感 |

下面按"关卡节奏 → IAA → 社交裂变"三部分展开，每部分末尾附对照表说明具体落到了哪个原则。

---

## 1. 关卡节奏（跨关卡的长线曲线，区别于单关60s内的节奏）

### 1.0 总关卡量规划（先定数字，避免和开发阶段的验证批次混淆）

单关体验（60秒游玩+结算/重试）约1~1.5分钟，据此推算：

| 阶段 | 关卡量 | 用途 |
|---|---|---|
| 开发计划M2验证批次 | 10~20关 | **仅用于验证核心循环/数值曲线/解锁节奏是否顺畅**，不是给正式玩家消费的内容量 |
| **正式上线主线** | **60~80关**（约2~3小时内容） | 覆盖完整的机制解锁节奏+足够的"深水区"组合关，避免玩家一次坐下就通关导致次日无理由回访 |
| 30关后 | 切换至**无限模式** | 复用同一套数值/道具/事件规则，程序化随机拼装难度递增关卡，用少量人力换取"打不完"的持续内容，是长线留存的兜底 |
| 上线后运营 | 每周/双周追加10~20关手工内容 | 维持"有新内容"的回访驱动力，配合1.4节的每日挑战轨道 |

> 20关作为**验证批次**是合理的；但如果直接当成上线总量，机制刚解锁完游戏就结束了，1.2节的错峰解锁本身就要用满20关，没有留出"深水区"支撑排行榜攀比/每日挑战这些长线钩子，会显得内容空。

### 1.1 难度曲线：锯齿状而非线性递增

不能一路变难（会持续挫败），也不能一路变易（会腻）。采用 **"3难+1易+1检查点"** 的5关循环：

```
第1关 →难度+  第2关 →难度++  第3关 →难度+++（小高峰）
第4关 →难度-（甜点关，刻意放松，给一次轻松通关的正反馈）
第5关 →检查点关（引入新道具/新机制，难度中等但有新鲜感）
↓ 循环继续，整体难度基线随组数缓慢爬升
```

### 1.2 内容解锁节奏（错峰释放，制造"还有更多没见过"的期待）

| 关卡区间 | 解锁内容 |
|---|---|
| 1~5 | 仅开放"加需求+改需求"，教会玩家分辨白卡/红卡 |
| 6~10 | 解锁"丢锅"，引入队列堆积→范围清空的场景 |
| 11~15 | 解锁"拍马屁"能量槏，引入"攒大招"节奏 |
| 16~20 | 引入"老板临检"机制，首次体验强制结算的风险 |
| 21~30 | 组合难度：多机制叠加，进入"主线深水区"，全部道具/事件已解锁 |
| 31~80 | 深水区延伸：难度曲线继续锯齿爬升，穿插专属挑战三星目标（见核心玩法策划6.2），维持主线厚度 |
| 80+ | 进入**无限模式**（见1.0），程序化生成，难度持续递增，作为主线之后的长期兜底内容 |

### 1.3 主题小节与叙事包装

每10关绑定一个叙事阶段（"入职第1周"→"季度考核"→"转正倒计时"），配合关卡列表UI的场景变化（办公室背景逐渐"升级"），给玩家阶段性的仪式感和成就感，而不是无差别的"第N关"列表。

### 1.4 每日挑战轨道（独立于主线进度）

- 每天用当日日期做随机种子生成一个**限时挑战关**（不同数值组合），24小时内可重复挑战刷新分数，次日重置。
- 奖励为限定道具/货币，不影响主线进度——目的是给"今天要不要打开游戏"一个具体、每天更新的理由，是最基础的**日活钩子**。

### 1.5 失败-重试的顺畅性

- 失败后**不重新加载整关**，直接停留在结算弹窗，点"重试"0.5秒内回到关卡开局，不打断心流。
- 连续失败2次后，弹出"要不要降低本关难度"的软性提示（而非强制），防止卡关流失，同时不剥夺想硬啃的玩家的选择权。

**原则对照**：锯齿曲线→上头；错峰解锁→上头；失败重试免加载→顺畅；软性降难度提示→顺畅（防流失）。

## 2. IAA 广告变现方案

微信小游戏的变现主力是**激励视频（用户自愿）**，插屏只作为补充且必须严格控频——广告体验和留存是直接对立关系，必须精细设计。

### 2.1 激励视频广告点位（自愿换奖励，对体验伤害最小）

| 场景 | 触发时机 | 奖励 |
|---|---|---|
| 复活/重试 | 失败结算弹窗，**每关限1次** | **明确定义**：失败判定只发生在认可度**=100**（或Boss临检结算后≥100，见核心玩法§5.4）这一种情形，复活时认可度回滚至**危险区下限**（数值由 `assets/config/balance.json` 的 `zones.danger.lo` 驱动，M4调优后当前为 **70**），同时倒计时**额外增加8秒**，并保证复活后传送带上不会立即出现导致二次瞬间死亡的Boss结算（若复活时Boss卡仍在场上，本次临检直接作废，重新计时下一次出现）。回滚目标是"给一次真实的翻盘机会"，不是"原地满血复活后立刻再死一次"。**M2路径B已落地**：`Game.revive()` / `ApprovalSystem.revive()` / `ConveyorSystem.clearBoss()` |
| 双倍奖励 | 通关结算弹窗 | 星级奖励/金币翻倍 |
| 应急回槏 | 丢锅/拍马屁能量槏不足、且场上白卡堆积告急时 | 立即回满该道具能量槏（仅限能量槏类，CD类不做广告加速，避免破坏资源节奏的策略性） |
| 额外宝箱 | 每日签到/每日挑战结算后 | 多开一个随机奖励宝箱 |

### 2.2 插屏广告（被动展示，严格限频）

- **只在自然停顿点插入**：返回关卡列表页时，或每完成3关触发一次——绝不在游玩过程中、结算高光动画播放期间弹出。
- **新手保护期**：前3关完全不插广告，先让玩家体验到核心乐趣再考虑变现，避免第一印象被打断导致次日留存暴跌。
- **频次上限**：单次会话插屏不超过3次，两次间隔≥90秒。

### 2.3 可选IAP补充

- "免广告月卡"/"去广告特权"：给愿意付费的用户一条不看广告的路径，双轨制变现，不强迫所有用户都走广告路线。

**原则对照**：激励视频自愿领取→顺畅+爽感（奖励即时可见）；插屏严格限频+新手保护期→顺畅（不打断心流）；应急回槏场景→爽感（解了危机的即时满足）。

## 3. 社交裂变方案（微信生态原生玩法）

### 3.1 求助式分享（最经典裂变钩子）

- 卡关/失败时提供"分享给好友求援"按钮：分享后获得一次额外复活或道具补给。
- 文案走**职场共鸣梗**路线（"我被AI逼到墙角了，救救我""这个月KPI快被AI抢完了"），本身具备传播性，不需要硬广式文案。
- **必须是可选加成，不能阻断主流程**（微信规则红线，见3.4）。

### 3.2 好友排行榜（微信开放数据域）

**指标必须重构为"越大越好"的正向计数，不能用"压制分越低越好"这种反直觉指标**（人晒战绩的本能是数字越大越牛，"我压在23分"vs"我压在19分"比的是谁更小，社交货币属性弱，对比"羊了个羊"的"我通关了"是清晰的正向成就）。排行榜展示维度改为：

- **职场段位**（综合表现分级，**口径**：由「累计猎杀式通关次数×3 + 累计三星关卡数×1 + 当前最高解锁关卡数×0.5」加权求和后分档，权重向"猎杀次数"倾斜以呼应§3.3的核心传播资产，具体系数在M4测试阶段用真实分布校准段位区间是否合理）：

| 段位 | 加权分区间（示例，待校准） |
|---|---|
| 实习生 | 0~20 |
| 打工人 | 21~50 |
| 卷王 | 51~100 |
| 反卷斗士 | 101~200 |
| AI克星 | 200+ |

- **AI累计崩溃/被劝退次数**：对应核心玩法§6.2①的猎杀式通关次数累计，次数越多越牛，正向计数。
- **存活天数叙事**："入职第N天"——用最高关卡数包装成天数叙事，数字越大越有梗。

结算页直接可见"你比XX好友段位高一级 / 多让AI崩溃了几次"，把排行榜数据变成即时反馈的一部分，而不是一个要专门跳转才能看到的孤立页面。

### 3.3 战报分享（社交货币化，重构为正向徽章制）

- 结算页自动生成一张**meme化战报卡片**，指标统一采用3.2的正向计数体系，例如：
  - 猎杀式通关战报："入职第7天，我把AI逼到当场崩溃被劝退，段位：反卷斗士"
  - 生存式通关战报："入职第7天，死死扛住了AI的KPI攻势，段位：卷王"
  - 标签化梗文案叠加使用（"本月KPI被我一个人扛下了"），进一步强化"越大越猛"的可炫耀属性。
- 这是本作**最大的传播资产**——"别让AI替代你"本身命题就贴职场热点，指标重构为正向徽章后，战报设计得越有梗、越想让人截图，裂变效率越高。

### 3.4 群聊挑战（改为轻量常驻入口，而非仅节点性活动）

**修正**：不强制分享≠不能做群机制，"羊了个羊"式的真正爆发恰恰来自群/队/朋友圈团战，此前"不建议常驻"的判断偏保守，调整为：

- 保留一个**始终在线的轻量群排行/群挑战入口**（自愿参与，不阻断主线，不违反3.5合规红线），玩家随时可生成"挑战链接"发到群里，群内好友玩同一数值的挑战关、实时排名比拼。
- **社交轨道专属的"墙"设计**：主线关卡（1.0节）刻意采用锯齿曲线避免持续挫败，但群挑战/每日挑战可以放"高难度、高分享欲"的数值，作为"羊了个羊式的墙"落地位置——不必在主线牺牲顺畅去建墙，把爆款需要的高挫败点转移到这个独立、自愿的轨道。但要注意**难度上限**：墙要用"步数/时限"卡人（如"限时90秒内挑战""固定3次机会"），而不是纯数值碾压卡人，保证大多数参与者能摸到及格线再往上肝分——否则排行榜/战报会全是低分素材，反而伤害3.2/3.3的传播效果。
- 平时节点性运营活动（周末限时挑战）可以复用同一入口叠加更强的专属奖励，两者不冲突。

### 3.5 合规红线（必须遵守）

- **严禁强制分享**：分享入口必须全部是"锦上添花"的可选加成，任何核心内容（如进入游戏、领取主线关卡奖励）不能以"必须分享才能获得"为前提，否则违反微信小游戏平台规则，有下架风险。
- 分享文案不能诱导用户执行分享之外的行为（如诱导关注公众号），保持在"分享得福利"的单一合规模式内。

**原则对照**：求助分享→上头（社交绑定回访动机）；排行榜→上头（长期攀比驱动力）；战报卡片→爽感（晒战绩的满足感）+ 自然传播；合规红线→顺畅（不阻断主流程，不因违规被下架而中断体验）。

## 4. 三部分的协同关系（一图看懂）

```
关卡节奏（难度锯齿+内容解锁）─── 制造持续的"上头"驱动力
        │
        ├── 在自然的失败/停顿点 → 插入IAA（激励视频为主，插屏严控频次）→ 变现但不伤"顺畅"
        │
        └── 在自然的成功/结算点 → 触发社交裂变（战报/排行榜/求助分享）→ 传播且强化"爽感"
```

三者不是三个独立模块，而是**共享同一套"关卡结算流程"作为触发时机**——结算页同时承载：星级反馈（爽感）→ 双倍奖励广告位（IAA）→ 战报分享卡片（裂变）→ 下一关预告（上头钩子），一次停顿点打包完成三件事，避免多次弹窗打断体验。

## 5. 后续建议

1. 先用第1节的锯齿曲线+错峰解锁排出前30关的具体数值表（覆盖开发计划M2的验证批次+全部机制解锁），验证通过后再批量产出到60~80关的上线量，其余靠无限模式兜底，不需要一次性手工做完全部关卡。
2. IAA点位建议在M4测试阶段（见开发计划）同步接入并做频次AB测试，用真实数据校准频次上限，而非拍脑袋定数字。
3. 战报分享卡片的视觉设计建议单独拉一次美术评审——这是整个裂变链路里最容易"翻车"（做得土气就没人分享）也最容易"出圈"（做得好玩就自然传播）的关键物料。
